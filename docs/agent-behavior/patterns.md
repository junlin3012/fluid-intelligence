# Codebase Patterns

> Proven patterns specific to this project. Agents should follow these when implementing.

---

## Project Structure

- **Infrastructure**: `deploy/` (Dockerfile, Dockerfile.base, cloudbuild*.yaml)
- **Scripts**: `scripts/` (entrypoint.sh, bootstrap.sh)
- **Config**: `config/` (mcp-config.yaml)
- **GraphQL ops**: `graphql/` (organized by domain: customers, orders, products, etc.)
- **Docs**: `docs/` (architecture, runbook, agent-behavior, research, specs, plans)
- **Schema**: `shopify-schema.graphql` at repo root, gitignored (98K lines, regeneratable)

## Deployment

- **Platform**: Google Cloud Run (asia-southeast1)
- **Secrets**: All sensitive values in GCP Secret Manager, injected as env vars by Cloud Run
- **Two-layer build**: Base image (Rust compile, ~20 min) + App image (config/code, ~60s)
- **Build configs**: `deploy/cloudbuild.yaml` (app), `deploy/cloudbuild-base.yaml` (base)
- **Dockerfile path**: `docker build -f deploy/Dockerfile .` (context = repo root)
- **Health checks**: auth-proxy returns 401 on `/health` (this is correct — auth is working)
- **Min instances**: 1 (avoid cold starts), max: 1 (in-memory auth state)
- **`--no-cpu-throttling`**: REQUIRED for child processes

## Authentication

- **Auth proxy**: mcp-auth-proxy (Go) on :8080, front door for all traffic
- **Google OAuth**: For browser-based clients (Claude.ai). Per-user allowlists via `GOOGLE_ALLOWED_USERS`
- **Password**: For CLI clients (Claude Code). Set via `AUTH_PASSWORD` secret
- **Admin JWT**: Generated by `bootstrap.sh` for backend registration. Short-lived (5 min)

## Process Management

- **Entrypoint**: `scripts/entrypoint.sh` — bash script with tini as PID 1
- **5 processes**: Apollo (Rust), ContextForge (Python), dev-mcp bridge, sheets bridge, auth-proxy (Go)
- **Startup order**: Apollo → ContextForge → bridges → wait for health → auth-proxy → bootstrap
- **Crash detection**: `start_and_verify()` checks PID after 2s, `wait -n` monitors all processes
- **Shutdown**: SIGTERM trap kills all PIDs

## ContextForge Specifics

- **Port variable**: `MCG_PORT` (NOT `PORT`, NOT `MCPGATEWAY_PORT`)
- **Host variable**: `MCG_HOST` (must be `0.0.0.0`, defaults to `127.0.0.1`)
- **Health endpoint**: `/health` (NOT `/healthz`)
- **Entry point**: Direct `main()` invocation — CLI script and `python -m` both broken after venv modification
- **Venv**: `/app/.venv/` — DO NOT pip/uv install into it

## Backend Registration

- **Endpoint**: `POST /servers` on ContextForge (NOT `/gateways`, NOT `/backends`)
- **Auth**: Bearer JWT in Authorization header
- **Request body**: `{"server": {"name": "...", "url": "...", "transport": "sse"}}`
- **Transports**: `sse` for all backends (Apollo, dev-mcp, sheets). ContextForge's MCP client has a bug with `streamablehttp` transport — use `sse` until fixed.
- **Idempotent**: 409 = already exists, treated as success

## Secrets & Keys

- All secrets in Secret Manager, never hardcode or commit
- Secret names: `shopify-client-id`, `shopify-client-secret`, `mcp-auth-passphrase`, `mcp-jwt-secret`, `google-oauth-client-id`, `google-oauth-client-secret`, `google-sheets-credentials`, `db-password`

## Scaling

- **max-instances=1**: Required while mcp-auth-proxy uses in-memory state
- **Upgrade path**: Move auth state to Redis/PostgreSQL, then increase max-instances
- **Honest scale**: Designed for 2-5 users. Document upgrade path, don't claim untested scale.
