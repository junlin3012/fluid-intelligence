2 Three Sources of Skills

Skills come from three places. Your Mirror Polish skill is a Personal Skill — you created it, you own it, you can edit it anytime.

✍

Your Personal Skills

~/.claude/skills/
  • mirror-polish-protocol
  • You create, edit, delete
  • Available to ALL projects
  • Plain markdown files
  • Survives forever
🔌

Plugin Skills

~/.claude/plugins/cache/
  • superpowers (15 skills)
  • frontend-design (1 skill)
  • Community maintains them
  • Auto-updated by plugins
  • Read-only for you
📁

Project Instructions

your-repo/CLAUDE.md
  • Per-repo rules & context
  • Checked into git
  • Team can see & edit
  • Only for THIS project
  • Not technically "skills"

4 How Claude Finds Your Skill

When you ask for something, Claude checks if a skill matches. Here's the exact flow for your Mirror Polish skill.

You say something

"Do an exhaustive code review" / "Find all bugs" / "Polish this codebase"

Claude reads ~/.claude/CLAUDE.md at conversation start

Sees the skills table — matches your request to a trigger rule:
"Exhaustive code review? → mirror-polish-protocol"

Claude loads your skill

Reads the full SKILL.md file from:
~/.claude/skills/mirror-polish-protocol/SKILL.md

Claude follows the skill instructions

Runs the Mirror Polish Protocol: brainstorm angles → review → triage with systematic debugging → TDD fix → write report → repeat until clean

Key insight: The skill is just a markdown file. Claude reads it and follows it like instructions you typed yourself. There's no magic — it's a document that shapes Claude's behavior.

Copy the SKILL.md file to anyone, or push to a git repo

rm -rf ~/.claude/skills/mirror-polish-protocol/
Bottom line: Everything is plain markdown files on YOUR computer. You own them. Claude reads them. There is no cloud, no lock-in, no magic. Edit a file, and Claude's behavior changes next conversation.
